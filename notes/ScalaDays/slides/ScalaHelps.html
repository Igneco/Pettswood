Why is this easier in Scala?

Terseness. Fixture code is much, much smaller, so it is easier to read and maintain. Pettswood has only 470 lines of production code (and 921 lines of test code).
  • Built-in XML processing. HTML is just a subset of XML (except for where it is horribly broken, but that is another topic)
  • First-class functions, such as declaring probe classes in the MultiRow table.
  • Anonymous inline functions, such as adding Concepts to the DomainBridge in Mixins
  • Pattern matching, especially when applied to XML parsing
  • Case classes, for short, simple Probes
  • Object-oriented, for the bits that aren't sequence transformations
